# Claudeによる指摘

## 用語の正確性について

記録に「ブロックのインスタンスに`life` という属性を与えることで」とありますが、実際に実装したのは`status`という属性です。

- **実装**: `this.status = 1; // 1: アクティブ, 0: 破壊済み`
- **参照**: [block.js:8](../js/block.js#L8)

`life`という名前も良い選択肢ですが、今回は以下の理由で`status`を選択しました：

1. **状態を表す汎用性**: 「生存/破壊」だけでなく、将来的に「無効」「凍結」などの状態も表現可能
2. **一般的な命名規則**: ゲーム開発では`status`が広く使われている
3. **拡張性**: 数値で表現することで、`status = 2`（2回耐久）なども実装しやすい

`life`は「耐久力の残り回数」を表すのに適しているため、将来的に「何回当たったら壊れるか」を実装する際に使用するとよいでしょう。

例：
```javascript
this.life = 3;  // 3回当たったら壊れる
this.status = 1; // アクティブ/非アクティブ
```
